# AllMem

跨 AI 工具的统一记忆管理，让 AI 不再在每个对话里从零认识你。

你在 Claude Code 里讨论了半天架构，换到 Codex CLI 又得从头说起；同一个项目开了无数对话，之前踩过的坑 AI 全忘了。AllMem 自动提取、整理、归档你在各个 AI 工具中的对话记忆，让 AI 从第一句话就了解你和你的项目。

## 这版适合立刻发帖的点

- **跨工具统一记忆**：扫描 Claude Code / Codex CLI 的本地对话日志，整理成项目长期记忆和近期动态
- **三智能体管线**：Narrator 提取因果链，Curator 负责 WAL + 长期压缩，Distiller 蒸馏可复用经验
- **经验到技能候选**：经验页可以把高价值经验提升成 skill candidate，更适合展示和后续自动化
- **一键加载演示数据**：没有私人日志也能马上看到完整界面，适合 GH 传播和课程演示
- **本地优先**：所有数据存在 `~/.allmem/`，无云端依赖

## MVP 当前包含什么

1. **概览页**
   - 同步真实日志
   - 一键加载 / 移除演示数据
   - 展示三智能体管线和项目入口
2. **项目页**
   - 查看项目长期记忆、近期动态、历史版本
   - 导入文件 / URL 到项目记忆
   - 查看与项目相关的经验
3. **经验 / 技能页**
   - 搜索和筛选经验
   - 将经验提升为技能候选
   - 查看触发条件、步骤、验证方式
4. **设置页**
   - 配置 LLM
   - 安装 `/allmem` / `/allmem-sync` / `/allmem-undo` skill

## 为什么做“一键演示数据”

GH 项目早期最大的阻力通常不是功能不够，而是体验门槛太高。很多人 clone 项目后没有你的私人日志，界面一打开就是空的。

这版直接提供一套演示工作区：

- 示例项目记忆
- 示例近期动态 timeline
- 示例经验卡
- 示例技能候选卡

这样别人第一次打开就能理解 AllMem 的价值，也更容易截图、录屏、发帖讨论。

## 开发者快速开始

```bash
# 依赖：Node.js >= 18, Rust >= 1.75
cd allmem
npm install
npx tauri dev
```

首次编译 Rust 约 2-5 分钟，之后会自动弹出桌面窗口。

## 使用方式

### 方式 1：体验演示版

1. 打开 AllMem
2. 在概览页点击 `一键加载演示数据`
3. 进入“项目”页和“经验 / 技能”页查看完整演示

### 方式 2：同步真实日志

1. 在概览页点击 `同步真实日志`
2. AllMem 会扫描本地 Claude Code / Codex CLI 日志
3. 在项目页查看长期记忆、近期动态、版本历史
4. 在经验页查看蒸馏出的经验，并手工提升为技能候选

### 方式 3：注入上下文到 AI 工具

在设置页安装 skill，然后在 Claude Code / Codex 中使用：

- `/allmem`：加载项目记忆到当前会话
- `/allmem-sync`：保存当前对话到记忆
- `/allmem-undo`：撤销注入的上下文

## 技术栈

- 前端：React 18 + TypeScript + Tailwind CSS + Zustand
- 后端：Tauri 2（Rust）
- LLM：OpenAI 兼容接口（可自行配置 API Key 和模型）
- 存储：纯本地文件系统 `~/.allmem/`

## 这版不打算解决什么

- 暂不强做完整 skill 自动生成
- 暂不引入重型训练或 RL 流程
- 暂不支持更多 AI 工具的数据源

这版的目标很明确：先把“跨工具记忆 + 经验蒸馏 + 技能候选展示”做成一个足够清晰、足够能传播的产品版本。

## 未来方向

- 更强的 typed memory object 管理
- 更自动化的 experience -> skill 转化
- 更严肃的评测闭环和可复现实验
- 更多 AI 工具支持（Cursor、Gemini CLI 等）
